Lessons Learned

HTML and CSS

From my understanding, HTML files and CSS files are usually separated into their own folders in the directory so that the CSS files can be referenced in the HTML file, making the code easier to read, and more importantly allowing the CSS file to be reused in other files.
However, there are cases when CSS code can be used in the same HTML file, for specific use cases. Kind of like containers for a class in object oriented programming.
In my case, I used the in-file CSS because I was trying to figure out how it works. Also, I’ll need to figure out how to alter Notion’s HTML code before it would be plausible to create a dedicated CSS folder or files, since I’ll be recording most of the projects in, and exporting from, Notion.
So much to learn and work on and I truly am excited for it.

Git Branches

To be honest, I didn’t do much research behind how the GitHub repository works. However, I did learn a little more about branches.
First there’s the main branch, where all of the final code will be committed and merged from other branches.
Second, I began using branches as a way to edit code without tampering with the original main branch. I actually created a branch for a branch during this project; one for the HTML code integration, the second for the CSS code integration. Since all is working correctly I can merge the second branch into the first branch and continue editing the code in there. Then once I’m done editing the HTML code I can merge the HTML branch into the main branch.
